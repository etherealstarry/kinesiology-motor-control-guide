## 四、不要犯的错（重要！）

### ❌ 错误 1：MkDocs 链接路径写错（最常犯！）
**错误写法**：
```markdown
[课程笔记](./必修课程/Motor-Neuroscience-I.md)  ← ❌ 不能写 `./` 开头
[核心课程](必修课程/)  ← ❌ 不能写 `目录名/`（MkDocs 无法识别）
[回首页](../index.md)  ← ❌ 层级算错
```

**正确写法**：
```markdown
[课程笔记](必修课程/核心课程/Motor-Neuroscience-I.md)  ← ✅ 相对路径，去掉 `./`
[核心课程](必修课程/核心课程/index.md)  ← ✅ 目录必须指向 `index.md`
[回首页](index.md)  ← ✅ 同一目录下直接写文件名
[回首页](../index.md)  ← ✅ 上一级目录（每多一级就多一个 `../`）
```

**MkDocs 链接规则总结**：
1. 不能写 `./` 开头
2. 不能写 `目录名/`（必须写 `目录名/index.md`）
3. 相对路径层级算法：`../` 每多一级就多一个 `../`

### ❌ 错误 2：mkdocs.yml 中 nav 配置引用不存在的文件
**现象**：`mkdocs build` 时出现警告或错误
**解决**：注释掉 `nav:` 配置，让 MkDocs **自动生成**导航

### ❌ 错误 3：忘记 commit + push
**现象**：本地改了，但 GitHub 仓库和网站没有更新
**解决**：每次改动后，必须执行：
```bash
git add -A
git commit -m "描述本次改动"
git push origin main
mkdocs gh-deploy --force
```

### ❌ 错误 4：直接修改 `gh-pages` 分支
**现象**：部署失败，或网站显示旧内容
**解决**：`gh-pages` 分支由 `mkdocs gh-deploy` 自动管理，**不要手动修改**

### ❌ 错误 5：没有测试就直接部署
**现象**：网站显示 404 或样式错乱
**解决**：每次 build 后，先本地预览：
```bash
mkdocs serve  # 在 http://127.0.0.1:8000 预览
```
确认无误再部署。

## 八、常见问题（FAQ）

### Q1：如何添加新页面？
**A**：
1. 在 `docs/` 下创建新的 `.md` 文件
2. 在文件中添加 YAML 头（--- 分隔）
3. MkDocs 会自动生成导航（如果注释了 `nav:` 配置）
4. 如果手动配置了 `nav:`，需要在 `mkdocs.yml` 中添加新页面的路径

### Q2：如何添加图片？
**A**：
1. 将图片放在 `docs/assets/images/` 目录下
2. 在 Markdown 中引用：`![图片描述](assets/images/图片文件名.png)`
3. 提交并推送（图片会被 Git 跟踪）

### Q3：如何修改网站主题颜色？
**A**：在 `mkdocs.yml` 中修改：
```yaml
theme:
  name: material
  palette:
    primary: indigo  # 主色调
    accent: deep orange  # 强调色
```

### Q4：如何添加数学公式？
**A**：Material for MkDocs 已支持 MathJax，直接用 LaTeX 语法：
```markdown
$$
E = mc^2
$$
```
